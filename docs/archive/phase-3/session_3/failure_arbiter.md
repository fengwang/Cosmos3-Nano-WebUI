- **Evidence:** `webui/vitest.config.ts` `include` = `design-system/**`, `lib/**`,

  source files, not a webui test location or the runner config.
- **Resolution (per arbiter: stop and update the contract/spec first):** surfaced
  the fork to the owner (add-tests-and-widen vs strict-radius). Owner chose to add
  co-located specs + broaden the `include`. Recorded as the **UX-S3-A1** blast-radius

  the session-contract YAML is outside this session's write-radius. Then proceeded.
